Краткое описание:
Данный вариант использования описывает добавление информации о билетах работником системы.
Основной поток событий:
Данный вариант использования начинает выполняться, когда работник системы добавляет билеты в систему.
1. Создается новое мероприятие указывается :
- название спектакля; 
- дата;
- время; 
- место проведения; 
- тип билета; 
- зрительский ряд;
- зрительское место;
- цена билета;
- cтатус билета;
2. Данные отправляются в систему и там сохраняются


Альтернативные потоки:
Некорректная информация по мероприятию:
 при указании некорректной информации по мероприятие при добавлении информации по билетам система выводит сообщение об ошибке, 
 пользователь может ввести данные заново.

Предусловия:
Работник системы авторизован
Постусловия:
Если вариант использования выполнен успешно, в систему добавляется информация по билетам.
